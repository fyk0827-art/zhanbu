# 问答平台 — 本地部署文档

本文说明如何在 Windows / macOS / Linux 本机完整跑通：**MySQL + Java 后端 + React 前端**，并完成答题、模拟支付、跳转报告平台。

相关文档：[支付对接文档](./支付对接文档.md)

---

## 一、项目结构

```
问答平台/
├── app/                 # 前端（React + Vite），默认端口 3000
├── qa-backend/          # 后端（Spring Boot 3 + JPA），默认端口 8080
└── docs/                # 文档
```

| 组件 | 技术 | 本地地址 |
|------|------|----------|
| 用户/管理前端 | Vite + React | http://localhost:3000 |
| 业务 API | Spring Boot | http://localhost:8080/api |
| 数据库 | MySQL 8.x | localhost:3306 / `qa_collector` |

> `app/api/` 下另有 Hono 示例服务，**日常开发以 Java 后端为准**；前端通过 `VITE_API_BASE_URL` 指向 `8080`。

## 三、MySQL 准备

### 3.1 安装并启动 MySQL

确保本机 MySQL 服务已启动（Windows 服务、或 `brew services start mysql` 等）。

### 3.2 创建数据库

使用 root 或具备建库权限的账号登录，执行：

```sql
CREATE DATABASE IF NOT EXISTS qa_collector
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

### 3.3 表是否会自动创建？

**会。** 后端 `application.yml` 中配置了：

```yaml
spring.jpa.hibernate.ddl-auto: update
```

首次启动 Spring Boot 时会自动建表/更新表结构，并执行 `DataInitializer` 写入默认数据（管理员、年龄段、示例题目、系统设置等）。

**注意**：Hibernate **不会**自动创建「数据库名」，只会在已存在的 `qa_collector` 库中建表。

## 四、后端配置与启动

### 4.1 修改数据库连接

编辑 `qa-backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/qa_collector?useSSL=false&serverTimezone=UTC
    username: root          # 改成你的 MySQL 用户名
    password: 123456        # 改成你的 MySQL 密码
```

### 4.2 配置报告平台（模拟支付必配）

同一文件中 `partner` 段：

```yaml
partner:
  api-base-url: http://39.97.224.240:8848
  webhook-secret: <向报告平台索取的密钥>   # 不能为空，否则 confirm-payment 失败
  standard-amount-cents: 1900
```

密钥示例见 [支付对接文档](./支付对接文档.md)。本地需能访问外网 `39.97.224.240:8848`。

### 4.3 默认管理员（首次初始化）

| 项 | 值 |
|----|-----|
| 用户名 | `admin` |
| 密码 | `admin123` |

可在 `application.yml` 的 `admin.username` / `admin.password` 修改（仅影响首次 `DataInitializer` 创建账号的逻辑，已有数据时以数据库为准）。

### 4.4 启动方式

**方式 A：命令行（需已安装 Maven 并加入 PATH）**

```powershell
cd "c:\Users\liuziqi\Desktop\问答平台\qa-backend"
mvn spring-boot:run
```

**方式 B：IDE**

1. 用 IntelliJ IDEA 打开 `qa-backend` 目录  
2. 找到 `com.qacollector.QaCollectorApplication`  
3. 右键 → Run  

启动成功日志中应出现类似：`Tomcat started on port(s): 8080`。

### 4.5 验证后端

浏览器或 curl：

```text
http://localhost:8080/api/age-groups
```

应返回 JSON 年龄段列表。若连接数据库失败，检查 MySQL 是否启动、账号密码、库名是否正确。

## 五、前端配置与启动

### 5.1 安装依赖

```powershell
cd "c:\Users\liuziqi\Desktop\问答平台\app"
pnpm install
```

若无 pnpm：`npm install -g pnpm` 或改用 `npm install`。

### 5.2 配置 API 地址（可选）

默认已指向本地 Java 后端，一般**无需**改：

```ts
// app/src/services/api.ts
VITE_API_BASE_URL || "http://localhost:8080"
```

如需显式指定，在 `app` 目录创建 `.env`：

```env
VITE_API_BASE_URL=http://localhost:8080
```

### 5.3 启动开发服务器

```powershell
cd "c:\Users\liuziqi\Desktop\问答平台\app"
pnpm dev
```

浏览器打开：**http://localhost:3000**

> Vite 配置端口为 `3000`（见 `app/vite.config.ts`）。

## 八、功能自测清单

### 8.1 用户答题

1. 打开 http://localhost:3000 → 开始测试  
2. 输入年龄 → 答题 → 查看结果  
3. 题目数量由管理后台「系统设置」控制（默认 5 道），**服务端强制**，改前端无效  

### 8.2 管理后台

1. 打开 http://localhost:3000/admin  
2. 登录 `admin` / `admin123`  
3. 可管理：题目、年龄段、统一定价、**系统设置**（答题数量、支付模式 mock）、答题记录  

### 8.3 模拟支付与报告跳转

1. 完成答题后点击支付  
2. 前端调用 `POST /api/payments/create` → `POST /api/payments/complete`  
3. 后端在 **mock** 模式下调用报告平台 `8848` 的 `confirm-payment`  
4. 浏览器自动跳转到返回的 `frontendUrl`（一般为 `http://39.97.224.240:8842/#/?orderId=...`）  

支付记录可在 MySQL 表 `payments` 中查看 `partner_frontend_url`。

## 九、生产构建（可选）

### 9.1 前端打包

```powershell
cd app
pnpm build
```

静态资源输出在 `app/dist/public`，需用 Nginx 等托管，并配置反向代理到 `8080` 的 `/api`。

### 9.2 后端打包

```powershell
cd qa-backend
mvn -DskipTests package
java -jar target/qa-collector-backend-1.0.0.jar
```

生产环境建议：

- 将 `ddl-auto` 改为 `validate` 或 `none`，用迁移脚本管理表结构  
- 修改 `jwt.secret`、数据库密码、管理员密码  
- 使用 HTTPS 与环境变量注入敏感配置

## 十、常见问题

### Q1：后端启动报 `Communications link failure`

- MySQL 未启动  
- `application.yml` 中用户名/密码错误  
- 未创建数据库 `qa_collector`  

### Q2：`mvn` 不是内部或外部命令

- 安装 [Maven](https://maven.apache.org/download.cgi) 并配置 `MAVEN_HOME`、`Path`  
- 或仅用 IDE 运行 `QaCollectorApplication`  

### Q3：前端页面空白 / 接口 404

- 确认后端已在 **8080** 运行  
- 浏览器 F12 → Network，看请求是否指向 `http://localhost:8080/api/...`  

### Q4：管理后台保存价格或设置返回 403

- 重新登录 `/admin` 获取新 token  
- 确认请求带 `Authorization: Bearer <adminToken>`（前端会自动从 `localStorage.adminToken` 读取）  

### Q5：支付失败 / 无法跳转报告页

- 检查 `partner.webhook-secret` 是否已填写  
- 本机能否访问 `http://39.97.224.240:8848`  
- 后端日志中 `Partner confirm-payment` 相关错误  
- 详见 [支付对接文档](./支付对接文档.md)  

### Q6：表没有自动出现

- 确认连接的是 `qa_collector` 库  
- 查看启动日志是否有 Hibernate / SQL 异常  
- 手动执行：`SHOW TABLES;`  

### Q7：端口被占用

| 端口 | 处理 |
|------|------|
| 8080 | 修改 `server.port` 或关闭占用进程 |
| 3000 | 修改 `app/vite.config.ts` 的 `server.port` |
| 3306 | 修改 MySQL 端口并同步 JDBC URL |

## 十二、默认数据说明

首次启动且表为空时，`DataInitializer` 会初始化：

- 多个年龄段（含统一价格，默认约 $9.99 展示价）  
- 管理员账号 `admin`  
- 各年龄段示例选择题  
- `app_settings`：`quiz_question_count=5`，`payment_mode=mock`  

重复启动不会重复插入年龄段/题目（有 `count() > 0` 判断）；设置项仅在 key 不存在时 seed。
